# Webpack / Turbopack branch â€” install checklist

Use this when the studio is mounted inside Next.js (or any Webpack host). Turbopack doesn't implement `require.context`, so the plugin ships a `codegen` CLI that scans `.help.md` files at dev/build time and writes a TypeScript module of `helpFiles`. The setup works for **both Webpack and Turbopack** â€” Next.js 16+ default is Turbopack, so support both raw loaders together.

**Every step has a skip condition.** Re-running on a wired studio? Check each gate first.

## 1. Install supporting dev-deps

**Skip per-package if already in `package.json` (devDependencies or dependencies):**

- `raw-loader` â€” Turbopack loader for `.help.md` â†’ string.
- `chokidar` â€” file watcher for `--watch` (CLI gracefully skips watch if missing).
- `concurrently` â€” run the CLI watcher alongside `next dev`.

Install only the missing ones:

```sh
pnpm add -D raw-loader chokidar concurrently
```

## 2. Add a `.help.md` loader rule in `next.config.{ts,mjs}`

**Skip if:** `next.config.*` already has `'*.help.md'` in `turbopack.rules` **and** a webpack rule with `test: /\.help\.md$/`.

Otherwise, add what's missing:

```ts
const nextConfig: NextConfig = {
  // ...existing config
  turbopack: {
    rules: {
      '*.help.md': { loaders: ['raw-loader'], as: '*.js' },
    },
  },
  webpack: (config) => {
    const helpMdRule = { test: /\.help\.md$/, type: 'asset/source' as const }
    for (const rule of config.module.rules) {
      if (rule && Array.isArray(rule.oneOf)) rule.oneOf.unshift(helpMdRule)
    }
    config.module.rules.unshift(helpMdRule)
    return config
  },
}
```

If `turbopack` or `webpack` keys already exist, **merge into them** â€” don't overwrite.

The `oneOf` unshift is required â€” Next.js wraps rules in `oneOf` groups and the rule won't fire from `module.rules` alone.

## 3. Add ambient module declaration

**Skip if:** any `.d.ts` in the project already declares `module '*.help.md'` (grep `\.help\.md` in `*.d.ts` files).

Otherwise, create at project root:

```ts
// help-md.d.ts
declare module '*.help.md' {
  const content: string
  export default content
}
```

Only strictly needed if `.help.md` is imported directly anywhere in user code; codegen-only setups don't require it, but the file is cheap to keep.

## 4. Wire codegen into `package.json` scripts

**Skip per-script if already present:**

- `predev: "sanity-plugin-md-notes codegen"`
- `prebuild: "sanity-plugin-md-notes codegen"`
- `dev` already wraps `sanity-plugin-md-notes codegen --watch` via `concurrently`

Otherwise patch in only the missing scripts:

```jsonc
{
  "scripts": {
    "dev": "concurrently -k -n help,next \"sanity-plugin-md-notes codegen --watch\" \"next dev\"",
    "predev": "sanity-plugin-md-notes codegen",
    "prebuild": "sanity-plugin-md-notes codegen",
  },
}
```

**Adjust input/output paths if the project's schemas aren't at the CLI defaults.** Defaults: scans `./sanity/schemas`, writes `./sanity/help-files.ts`. Override with flags:

```
sanity-plugin-md-notes codegen --in ./src/sanity/schemaTypes --out ./src/sanity/help-files.ts --watch
```

**Skip if the generated file already exists** at the resolved `--out`. Otherwise run once now:

```sh
pnpm exec sanity-plugin-md-notes codegen
```

## 5. Gitignore the generated file

**Skip if:** `.gitignore` already lists the help-files path (grep for `help-files`).

Otherwise add:

```
sanity/help-files.ts
```

(Adjust path if `--out` was overridden.)

## 6. Wire `helpPlugin` + `withHelpDefaultDocumentNode` in `sanity.config.ts`

**Skip the whole step if:** grep finds both `helpPlugin(` and `withHelpDefaultDocumentNode` inside `sanity.config.*`. Sub-parts skip independently â€” see below.

Target shape:

```ts
import { defineConfig } from 'sanity'
import { structureTool } from 'sanity/structure'
import { helpPlugin, withHelpDefaultDocumentNode } from 'sanity-plugin-md-notes'
import { helpFiles } from './sanity/help-files' // generated by codegen

export default defineConfig({
  // ...existing config
  plugins: [
    helpPlugin({ files: helpFiles, githubRepo: 'org/repo' }),
    structureTool({
      structure,
      defaultDocumentNode: withHelpDefaultDocumentNode(),
    }),
  ],
})
```

**Sub-step skip rules** when patching an existing config:

- **Import line already present?** Don't re-add. Extend existing imports rather than duplicating.
- **`helpFiles` already imported?** Reuse the existing import; check it points at the same `--out` path you configured in step 4.
- **`structureTool({...})` already in `plugins`?** Inject `defaultDocumentNode: withHelpDefaultDocumentNode()` into the existing call. If `defaultDocumentNode` already exists, **compose** instead of replacing:

  ```ts
  defaultDocumentNode: withHelpDefaultDocumentNode((S, context) =>
    /* their existing resolver body */
  )
  ```

- **`helpPlugin(...)` already registered?** Don't re-add. If `githubRepo` isn't set and the user wants the footer, patch the existing call.

**`githubRepo`** â€” hand-write `'owner/repo'`, or pass `pkg.repository`. The Vite auto-detect (`sanity-plugin-md-notes/git-repo`) **does not work on Webpack/Turbopack** â€” it relies on the Vite plugin to populate the value.
