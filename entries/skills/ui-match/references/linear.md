# Linear ticket — one per page

Runs automatically after classification (SKILL.md step 5). One issue per page containing all
deltas with **inline** screenshots. Uses the hosted
Linear MCP (`mcp__plugin_linear_linear__*`) — OAuth only, no API key. If the first tool call fails
with an auth error, run `mcp__plugin_linear_linear__authenticate`, have the user complete the
browser flow, then `complete_authentication`.

## 1. Resolve the target
```
list_teams        # pick/confirm the team (key like "BLU", or the site's team)
list_projects     # pick/confirm the project for this site
```
Ask the user which team + project if not obvious. (Reuse the same target on later pages in the
session without re-asking.)

## 2. Upload each screenshot (prepare → PUT → embed)
Order that works: **create the issue first** with a placeholder body (step 3) to get its key,
then upload each image, then `save_issue` to set the full description with the embeds.

Only upload the **actionable deltas** (missing + extra + different). Downscale first to keep the
ticket light:
```bash
$SKILL_DIR/scripts/crop.sh scale <src.png> /tmp/ui-match/$SLUG/tkt-<name>.png 1000
```
Per image (signed URL lives ~60s — do one image's prepare → PUT immediately, one at a time):
```
prepare_attachment_upload{ issue:"<KEY-123>", filename:"tkt-x.png",
  contentType:"image/png", size:<exact bytes>, title:"<finding title>" }
```
PUT the bytes to the returned `uploadRequest.url`. Send only the **signed** headers (the response's
`X-Goog-SignedHeaders` lists just `content-type` + `x-goog-content-length-range`); size must match
exactly:
```bash
SZ=$(stat -f%z /tmp/ui-match/$SLUG/tkt-x.png)
curl -s -o /dev/null -w '%{http_code}' -X PUT --data-binary @"/tmp/ui-match/$SLUG/tkt-x.png" \
  -H "content-type: image/png" -H "x-goog-content-length-range: ${SZ},${SZ}" \
  "<uploadRequest.url>"          # expect 200
```
Keep the `assetUrl` that `prepare` returned. **Do not call `create_attachment_from_upload`** — for
a clean ticket we embed inline only (`create_attachment_from_upload` would also add a duplicate
attachment card at the bottom). Just reference the bare `assetUrl` in markdown: `![<title>](<assetUrl>)`.
When you `save_issue`, Linear auto-rewrites each `uploads.linear.app` URL with a `?signature=` token
and renders it inline — that rewrite in the response confirms the image was accepted.

## 3. Issue content (keep it un-bloated)
Formatting rules (user preference — follow exactly):
- **No em dashes** anywhere in the ticket. Use a hyphen `-` or parentheses.
- **Short, readable title**: `<Page> - Figma/live site parity` (no counts, no punctuation clutter).
- In **Links**, each label is an **H4 with no colon** and the URL goes on the **line directly below**
  the label (not inline `Label: url`). `Summary` is an H4 too.
- **No emoji status dots** (🔴🟡⚠️✅) in ticket headers or summary — plain text only.
- **Summary is a bullet list**, not a `·`-separated line.
- **Do not add** the "Depth: existence-only…" note, or any similar filler sentence.
- Avoid `·` middots generally (they read as clutter) — use bullets or commas.
- **No trailing divider** (`---`) and **no "Generated by /ui-match" footer** at the end. End on the
  Matched list.

- **title**: `<Page> - Figma/live site parity`   (e.g. `Homepage - Figma/live site parity`)
- **description** (markdown — literal newlines, no em dashes, no emoji):
  ```
  ## Links

  #### Figma design
  <figmaUrl>

  #### Live site
  <liveUrl>

  #### Summary
  - N missing
  - M extra
  - K built differently
  - P matched

  ## Missing on the live site (build these)
  ### <title>
  <desc>
  ![<title>](<assetUrl>)
  ... (one per missing item)

  ## Extra on the live site (confirm intended)
  ### <title>
  <desc>
  ![<title>](<assetUrl>)

  ## Built differently
  ### <title>
  <desc>
  **Figma**
  ![figma](<assetUrl-figma>)
  **Live site**
  ![live](<assetUrl-site>)

  ## Matched
  <name>, <name>, <name>   (text only, no images)
  ```
Create/update with `save_issue` (create = omit `id`; it needs `team`, `title`, and takes `project`,
`description`). Two calls total: one to create (placeholder body → get the key for uploads), one to
set the full description. Markdown wants **literal** newlines (no `\n` escapes).

## 4. Record the ticket
- Print the ticket URL/key in the terminal.
- Add `"ticketUrl"` and `"ticketKey"` to `/tmp/ui-match/$SLUG.findings.json` as the record.
